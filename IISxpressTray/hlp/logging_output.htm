IISxpress provides a log output option for both the filter and server components. Normally you would have the logging switched off since it can use a large amount of disk space and also it slows the compression process down. However, if you need to track a problem with IISxpress or you would like to capture server activity over a period of time it is a useful feature.

The log files will be written to the Logs folder under the main IISxpress folder.

On Windows 2003, Vista and 2008 the filter logging feature may require additional configuration. The IISxpress web server plugin will run with the same priviledges as the application pool used to run the web site. If this account does not have permission to write to the IISxpress Log directory then you will not see any log files written to your disk. Usually this account is called NETWORK SERVICE, but if you have configured your application pool to run under another account then you must also give this account permission to write to the IISxpress log directory.
