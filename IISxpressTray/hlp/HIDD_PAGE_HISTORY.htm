Compression History

The compression history window shows the recent compression activity recorded by IISxpress. This window displays the time the response was compressed, the URI used by the client browser, the original size of the response, the compressed size and the compression ratio achieved.

The text is colored blue in the history window if the response was served from the compression cache.

If you select an item from the list the detail area at the bottom of the window will show more information about the request and the response. If you then right click on an item in the control a context menu will be displayed:

From this menu you can clear the history window, control the scrolling behavior and add exclusion rules based on the item selected. The supported rules are:

  • Exclude this Extension - exclude the extension of the currently selected item.
  • Exclude this Content Type - exclude the content type of the currently selected item.
  • Exclude Matching Content Types - exclude the content type family of the currently selected item. For example, if the item content type is image/jpg the added rule will be image/*.
  • Exclude this Direcory - exclude the selected item's directory.
  • Exclude this Direcory Tree - exclude the selected item's directory and all its sub-directories.
  • Exclude this Web Site - exclude the web site where the selected item is located.
  • Exclude this Client IP - exclude the IP address of the client.
  • Exclude this Client Subnet - exclude the subnet where the client is located. If the client IP address is 192.168.2.22 then the subnet will be treated as 192.168.2.
Details

The Details area shows extended information about a client browser request and the compressed response. When you select an item from the activity history list the URI, query string, hostname, client IP address, content type and browser type are displayed. For more information on the browser type string see the following section.

If the details information overflows its screen area hover the mouse over it to request a tooltip display the entire string.


Browser Identification

IISxpress will try and identify the client browser by analysing the User-Agent string it passes to IIS in the HTTP request. Currently IISxpress recognises the following browsers:

  • Internet Explorer 8.0, 7.0, 6.0, 5.5, 5 and 4. Earlier versions will be identified as MSIE.
  • Firefox 2.0, 1.5 and 1.0.x. Earlier versions will be identified as Firefox.
  • Mozilla 1.7.x and numerous earlier versions.
  • SeaMonkey 1.1 and 1.0.
  • Netscape 9, 8, 7 and 6. Earlier versions will be identified as Netscape.
  • Opera 9.0, 8.5, 8.0, 7 and 6. Earlier versions will be identified as Opera.
  • Safari 3.0, 2.0, 1.3, 1.2 and 1.0.

If IISxpress cannot identify the browser it will display the User-Agent string as it was sent by the client browser.
