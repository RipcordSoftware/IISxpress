All responses have a Content Type field. This field is defined in the HTTP/1.1 standards document, it contains a predefined string denoting the type of the data in the response. For example, responses containing plain text data would have a content type of text/plain.

IISxpress supports exclusion of responses from compression based on their content type. This rule type allows you to exclude classes of responses rather than just individual files by their extension.

Content type rules are especially useful if you have dynamically generated content where the type of the content can change depending on the parameters passed in the request. For example, an ASPX file may return BMP (uncompressed Windows bitmap) or JPG content depending on the request parameters. You may wish to compress the response if it contains uncompressed BMP data, and not compress if it contains JPG data.

The content type for JPG data is image/jpg, to exclude all JPG data you should add this as an exclusion rule.

Exclusion Rule Forms

IISxpress supports two rule forms for content type exclusion. You can specify the exact type, for example, image/jpg or alternately you can exclude all images from being compressed by specifiying a wildcard content type rule, for example image/*.
