The Content Types window displays the response content types which IISxpress will not compress. It is recommended that any responses containing pre-compressed data, data which will not compress well or where there is a known issue with the client browser accepting compressed data for this content type should be added to the list.

It is highly recommended that any streaming data like a live audio feed, webcam or other live video feed should be excluded from being compressed. Ask the appropriate software vendor for their content types (MIME types) and then add them to this list.

To add a content type to the list click the Add button. To remove content types from the list select the unwanted items and press the Delete button.
