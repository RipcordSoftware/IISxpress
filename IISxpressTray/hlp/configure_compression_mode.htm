IISxpress supports three different compression modes. They are Default, Best Speed and Best Size. You should choose the most suitable mode for your application.

  • Default mode offers the best compromise between speed and size. In most cases this is the mode to choose.
  • Best Speed mode is well suited to high load environments, it offers quick response times and respectable compression performance.
  • Best Size mode will attempt to compress the content to the smallest possible size regardless of CPU time used. It is the slowest setting.

The image above shows a comparison between the three compression modes. The results are listed in the order Default, Best Speed and Best Size. The Default mode is usually capable of performance approaching 1 to 2% of the Best Size mode.

You can change the compression mode on the Configuration Tab of the IISxpress Status Window.
