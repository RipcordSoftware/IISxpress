The IISxpress ISAPI Filter provides the interface between IIS and the IISxpress Server. When IIS starts it loads the IISxpress filter into memory, the filter then intercepts all responses and passes them to the IISxpress Server where the exclusion rules are applied, compression is performed and then the response is returned to the client.

For IISxpress to function the filter must be installed correctly in IIS. This will have been done during the IISxpress installation process. However if you experience problems you can check that the filter is installed using the following methods:

  • Open the IISxpress Status Window and select the General tab if it not already displayed. The status of the filter should be Installed.
  • If you are running Windows XP, Windows 2003 or better then you can run the IIS Administration plugin, right click on the 'Web Sites' item in the tree on the left of the main window and select the properties menu item. Now select the ISAPI Filters tab, IISxpress should appear at the bottom of the list of installed filters. When you click on IISxpress in the filter list the Details area of the window should show that the Status is Loaded.

The IISxpress filter should always appear last in the IIS filter list since this will be the last filter to run before the data is returned to the client.

If you continue to experience problems you can use the 'Repair' option on the Configuration tab of the IISxpress status window to reset the IISxpress filter installation to the default settings.
