The URI window displays the URIs that IISxpress will NOT compress. If you wish to prevent responses from certain URI paths from being compressed then you should add them to this list.

The URI exclusion rule works in two modes, 'Folder' and 'Branch'. In folder mode the path to the response must match the request URI path exactly. In branch mode IISxpress will match all sub-folders of the excluded URI.

The URI match rule is applied in the context of the web server instance that has handled the request. If you have one web server instance which responds to www.abc.com and another which uses www.xyz.com then you apply independent exclusion rules for each.

For example, to prevent the /streams URI path on web server instance www.abc.com from being compressed select the correct instance, select the streams item in the URI tree and right click to display the context menu. Select Exclude Folder from the menu. The streams item should now be displayed in red text indicating that it is excluded.

To remove the exclusion rule from a item in the tree simply select the excluded item, right click with the mouse to display the context menu and then select Include.
