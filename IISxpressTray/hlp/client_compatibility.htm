All browser based web traffic is enclosed in a protocol called HTTP. This standard includes support for three modes of compression - deflate (ZLIB), GZIP and compress. In reality deflate and GZIP are almost identical and also have the greatest browser support. IISxpress supports both deflate (ZLIB) and GZIP.

When a client browser makes a request to a web server it includes information which allows the server to determine whether the client can support compressed responses. This information is enclosed in a request parameter called 'Accept-Encoding'. IISxpress uses this information from the client to decide whether a response is a candiate for compression. If the client cannot support either deflate (ZLIB) or GZIP the response is not compressed.

The only modern browser which does not support compression is Internet Explorer 5.2 for Mac OS X. No responses destined for this browser will be compressed since it does not send a suitable 'Accept-Encoding' parameter.

Sometimes you may find an incompatibility or some undesirable behaviour, it is in these cases that you should use IISxpress' exclusion rule engine to maintain compatibility and your end users' experience.

We recommend that you test all popular browsers when you install IISxpress on your web server. You should seriously consider testing with: Internet Explorer, Firefox, Opera, Safari and Chrome.
