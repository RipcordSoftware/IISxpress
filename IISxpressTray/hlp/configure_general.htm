IISxpress Filter

The IISxpress Filter area shows the status of the IISxpress ISAPI filter and allows you to enable or disable response processing.

The IISxpress filter will only process responses when it is enabled. If the Allow filter to process responses checkbox is checked (the default) then IISxpress will process and compress responses. If the checkbox is not checked then IISxpress will not process any responses and IIS will return uncompressed data as normal. It is recommended to leave this option checked unless you are experiencing problems with IISxpress.


IISxpress Server

The IISxpress Server area shows the status of IISxpress Server, it allows you to start or stop the server, set the compression mode and reset the compression statistics.

Click the Start button to start the IISxpress Server and the Stop button to stop the IIS Server. When the service is stopped responses will not be compressed.

To change the compression mode select from Default, Best Speed or Best Size. The compression mode is changed instantly so there is no need to restart IISxpress.

In addition to the HTTP standard deflate and GZIP compression algorithms IISxpress also includes support for BZIP2 which is a highly efficient alternative. While BZIP2 compresses content to a much smaller size that deflate/GZIP it is also much slower and will therefore use more CPU resources. For this reason you can disable BZIP2 support if you wish. The alternate technique is to allow IISxpress to compress static content (HTML, CSS, JS, etc.) which can be stored in the compression cache resulting in it being compressed much less frequently reducing the CPU load but gaining the benefit of the enhanced compression. You can also elect to use BZIP2 for all content, for example ASP.NET, ASP, CGI, etc.

The default BZIP2 compression mode is 'Static Content'.

To reset the overall compression statistics click the Reset button. All global metrics will be reset.
